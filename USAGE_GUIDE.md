# ECO FITNESS - Руководство пользователя

## Обзор системы

Полнофункциональная платформа управления премиальным фитнес-клубом с тремя ролями:
- **Клиент** - запись на тренировки, покупка абонементов, отслеживание прогресса
- **Тренер** - управление расписанием, отметка посещений, работа с клиентами
- **Администратор** - управление пользователями, тренировками, статистика

## Полный User Flow

### 1. Регистрация и вход
1. Откройте главную страницу
2. Нажмите "Зарегистрироваться" или "Войти"
3. Введите данные (для теста используйте тестовые аккаунты)
4. После входа выберите роль на экране выбора роли
5. Вы будете перенаправлены в соответствующий кабинет

### 2. Клиент

#### Запись на тренировку
1. Перейдите в раздел "Расписание"
2. Выберите тренировку
3. Нажмите "Записаться"
4. Подтвердите запись в модальном окне
5. ✅ Вы записаны! Тренировка появится в "Мои записи"

**Важно:** 
- Для записи нужен активный абонемент
- Проверяется наличие свободных мест
- У базового абонемента ограниченное количество посещений

#### Отмена записи
1. Перейдите в "Мои записи"
2. Найдите тренировку
3. Нажмите "Отменить"
4. Подтвердите действие
5. ✅ Запись отменена, место освобождено

#### Покупка абонемента
1. Перейдите в раздел "Абонемент"
2. Выберите подходящий тариф:
   - **Базовый** (4 990₽) - 20 посещений
   - **Премиум** (7 990₽) - безлимит
   - **VIP** (14 990₽) - безлимит + персональные тренировки
3. Нажмите "Купить"
4. Подтвердите покупку
5. ✅ Абонемент активирован!

#### Редактирование профиля
1. Перейдите в раздел "Профиль"
2. Нажмите "Редактировать"
3. Измените данные
4. Нажмите "Сохранить"
5. ✅ Профиль обновлён

### 3. Тренер

#### Начать тренировку
1. Перейдите в "Расписание" или "Обзор"
2. Найдите запланированную тренировку
3. Нажмите "Начать тренировку"
4. Подтвердите действие
5. ✅ Тренировка отмечена как "Идёт"

#### Отметить посещение
1. Во время тренировки (статус "Идёт")
2. В списке участников нажмите:
   - ✅ - клиент пришёл
   - ❌ - клиент не пришёл
3. ✅ Посещение отмечено, создана запись в истории

#### Завершить тренировку
1. Нажмите "Завершить тренировку"
2. Подтвердите действие
3. ✅ Тренировка завершена

#### Просмотр клиентов
1. Перейдите в раздел "Клиенты"
2. Увидите всех клиентов по тренировкам
3. Отображаются статусы записей

### 4. Администратор

#### Управление пользователями
1. Перейдите в раздел "Пользователи"
2. Используйте поиск для фильтрации
3. Доступные действия:
   - 🔒 Заблокировать/разблокировать
   - 🗑️ Удалить пользователя

#### Блокировка пользователя
1. Найдите пользователя
2. Нажмите иконку блокировки
3. Подтвердите действие
4. ✅ Пользователь заблокирован (не может войти)

#### Удаление пользователя
1. Найдите пользователя
2. Нажмите иконку удаления
3. Подтвердите действие
4. ✅ Пользователь и все данные удалены

#### Просмотр статистики
1. На вкладке "Обзор" доступны:
   - Общее количество клиентов
   - Активные абонементы
   - Общий доход
   - Посещения за сегодня

#### Управление тренировками
1. Раздел "Тренировки"
2. Просмотр всех тренировок
3. Отображение статусов и загрузки

#### Управление абонементами
1. Раздел "Абонементы"
2. Список всех активных абонементов
3. Информация о клиентах и сроках

#### Управление акциями
1. Раздел "Акции"
2. Просмотр активных акций
3. Удаление акций

## Логика и валидация

### Проверка при записи на тренировку
- ✅ Наличие активного абонемента
- ✅ Срок действия абонемента
- ✅ Количество свободных мест
- ✅ Проверка на дубли (повторная запись)
- ✅ Количество посещений (для базового абонемента)

### Автоматические действия
- При записи: уменьшается количество мест, списывается посещение (базовый)
- При отмене: место освобождается, посещение НЕ возвращается
- При отметке посещения: создаётся запись в истории

### Состояния тренировок
1. **Запланирована** (scheduled) - можно записаться
2. **Идёт** (ongoing) - можно отмечать посещения
3. **Завершена** (completed) - архив
4. **Отменена** (cancelled) - все записи отменены

### Уведомления (Toast)
Все действия сопровождаются уведомлениями:
- ✅ Успешные операции - зелёные
- ❌ Ошибки - красные
- ℹ️ Информация - синие

## Технические детали

### Архитектура
- **AuthContext** - аутентификация, роли, вход/выход
- **AppContext** - все данные и действия
- **ProtectedRoute** - защита маршрутов по ролям
- **Модальные окна** - подтверждение всех критичных действий

### Основные компоненты
- `ClientDashboard` - 6 вкладок, полная логика клиента
- `TrainerDashboard` - 4 вкладки, управление тренировками
- `AdminDashboard` - 6 вкладок, управление системой

### Состояния загрузки
Все асинхронные действия имеют:
- Loading состояние (spinner)
- Success состояние (уведомление)
- Error состояние (уведомление с описанием)

## Возможности расширения

Для подключения реального backend:
1. Замените методы в `AppContext` на API вызовы
2. Добавьте обработку ошибок от сервера
3. Реализуйте JWT токены в `AuthContext`
4. Добавьте загрузку данных при монтировании

## Советы по использованию
